foundation devex
Span retention policy + DB health panel
Auto-prune spans by age (CLAUDESEC_RETENTION_DAYS, default 30) and count (CLAUDESEC_MAX_SPANS, default 50,000). GET /api/db-stats, POST /api/db-stats/prune, POST /api/db-stats/retention. DB health panel in Costs tab with usage bar and inline retention config editor.
observatory API
Behavioral anomaly detection
Statistical rules layer on top of regex rules. Detects: token spike (single span > 4× session avg), threat burst (≥3 threats in last 10 spans → HIGH), excessive tool calls (>100 per session), off-hours activity (before 6am). Runs per-session after every OTLP batch. Fires webhooks. Deduped by 5–60 min windows.
security alert
Shareable session HTML report
GET /api/sessions/:traceId/report — self-contained HTML with inline CSS, no external deps. Shows: session header (name, harness badges, date), summary cards (spans, HIGH/MED/LOW threats, input/output tokens), alerts table, full span table with severity coloring, duration, tool name, matched threat rule. Download button in sidebar per session.

9
Resilience, DX & Release Polish
Rate limiting, trace import, enhanced CLI subcommands, live activity sparkline, and npm publish workflow
Rate limiting + ingestion safeguards
Per-IP token-bucket rate limiter on POST /v1/traces (configurable via CLAUDESEC_RATE_LIMIT_RPS, default 50 req/s burst 200). Circuit breaker auto-pauses ingestion when DB exceeds 90% of max_spans. Span-count guard per request (max 500 spans/batch). Returns 429 with Retry-After header.
security resilience
Trace import — POST /api/import + UI file drop zone
POST /api/import accepts ClaudeSec JSON export or raw OTLP JSON (auto-detected). Re-runs threat detection on imported spans. UI drag-and-drop zone in the sidebar (or import button) to import a saved .json export file directly in the browser.
core observatory
Enhanced CLI — claudesec status / export / reset / open
Add subcommand routing to cli/init.ts: `claudesec status` (hits /api/health, prints counts+uptime), `claudesec export [file]` (saves /api/export JSON), `claudesec reset` (confirms + calls /api/reset), `claudesec open` (opens dashboard in browser). `claudesec init` remains the default with no args.
devex CLI
Live activity sparkline in dashboard header
Real-time mini chart in the dashboard header showing spans/min and tokens/min over the last 60 seconds. Backed by GET /api/activity (returns last 60 one-second buckets). Updated on every graph-update socket event. Rendered as inline SVG path — no extra chart library.
UI observatory

Also bumps the version in package.json from the tag. Includes a release-notes step that creates a GitHub Release from CHANGELOG.md.

10
Deep Inspection, MCP & Live Tail
Span annotations, SSE live tail stream, MCP server mode, unified settings UI, and per-harness profiling
Span annotations — user investigation notes
GET/POST/DELETE /api/spans/:spanId/annotations. SQLite annotations table with text, author (default "analyst"), createdAt. Annotation count badge on span node in graph. Annotation editor in span detail panel (inline textarea, save/delete). Annotations included in HTML session reports and JSON export.
UI observatory
SSE live tail — GET /api/tail + claudesec tail CLI
GET /api/tail streams newline-delimited JSON of every new span as it arrives (Server-Sent Events). Supports ?harness= and ?severity= filters. claudesec tail [--harness X] [--severity high] CLI subcommand that connects to the SSE stream and pretty-prints spans to the terminal with color-coded severity.
devex CLI
MCP server mode — AI agents query their own telemetry
POST /mcp endpoint implementing the Model Context Protocol (JSON-RPC 2.0 stdio transport over HTTP). Exposes tools: get_sessions, get_spans(traceId), get_alerts, get_health, search_spans(query). Lets Claude Code and other MCP-capable agents introspect their own activity. Documented in docs/mcp.mdx.
multi-harness observatory
Settings panel — unified dashboard config UI
New Settings tab (gear icon) in the dashboard. Sections: Retention (max spans, days), Rate Limiting (RPS, burst, max batch), Webhook (URL, threshold, test), Display (show/hide zero-severity spans, timeline default zoom). All settings read/write via existing /api endpoints. No page reload required.
UI devex
Harness profile sidebar — per-agent stats & health
GET /api/harnesses returns per-harness aggregates: total spans, sessions, HIGH/MED/LOW threat counts, total tokens in+out, first/last seen timestamps. New "Harnesses" tab in dashboard sidebar showing cards for each active harness with color, icon, counts, threat heatmap badge, and last-seen time. Drives the existing harnessFilter in the graph.
multi-harness observatory

Threat heatmap — hourly threat density grid
GET /api/heatmap returns a 7×24 matrix (dayOfWeek × hour) of { spans, threats } counts, computed from span startNano timestamps. New Heatmap tab in the dashboard: colored grid cells (green→yellow→red by threat ratio), hoverable cells showing exact counts, day/hour axis labels. Helps detect "threat spikes always happen Tuesday evenings."
security UI
In-graph node search & highlight
Ctrl+F / ⌘+F opens a floating search overlay in the graph canvas. Typed query filters node label, span name, and attributes in real-time. Matching nodes glow with a colored ring; non-matching nodes fade to 20% opacity. Up/Down arrows or Enter cycles through matches and centers (fitView) on each. ESC or ✕ clears the search and restores all nodes.
UI observatory
Session pinning & favorites
Add pinned BOOLEAN column to sessions table (default 0). PATCH /api/sessions/:traceId accepts { pinned: true/false }. Sidebar shows a star (⭐) button on each session; pinned sessions float to the top with a gold star badge and a subtle highlight. Pinned state survives server restarts. Max 10 pinned sessions enforced server-side.
UI devex
Semantic attribute enrichment in span detail panel
Span detail panel groups attributes by OTel semantic namespace with icons: gen_ai.* (robot icon, token gauge bars for input/output), http.* (globe icon, method+status badges), db.* (database icon), file.* (file icon), process.* (terminal icon), tool.* (wrench icon). Unrecognized attributes fall into a collapsible "Other" section. Human-readable labels replace raw attribute keys.
UI observatory

Intelligence, Replay & Theme
Session comparison, threshold alert rules, graph replay, OTEL Collector config generator, and dark/light theme
Session comparison — side-by-side diff view
GET /api/sessions/compare?a=traceId1&b=traceId2 returns per-session diffs: span counts, threat counts by severity, token in/out, avg duration, top tools, threat rules triggered. UI: Ctrl+click two sessions in sidebar to enter compare mode; a ComparePanel slides in showing two-column diff with delta badges (green/red).
UI observatory
Threshold alert rules — numeric trigger conditions
New threshold_rules table (id, name, metric, operator, value, window_min, enabled). Metrics: tokens_in, tokens_out, threat_count, span_count, high_threat_count. Evaluated after every OTLP batch per session. Fires webhook + creates alert. CRUD: GET/POST/DELETE /api/threshold-rules. Settings panel section for managing them.
security alert
Graph timeline replay / scrubber
A "Replay" button on the graph canvas animates spans appearing in chronological order (sorted by startNano). Play/Pause, 1×/2×/5× speed, progress scrubber bar. Spans fade in as they're "received". ESC resets to full view. Works on selected session or all spans.
UI observatory
OTEL Collector config generator
GET /api/collector-config returns a ready-to-use otel-collector-config.yaml (receivers: otlp grpc+http, processors: batch+memory_limiter, exporters: otlphttp to ClaudeSec). Download button in dashboard. docs/collector.mdx explains usage.
multi-harness devex
Dark / light theme toggle
System-aware theme respecting prefers-color-scheme. Sun/moon toggle in header, stored in localStorage. Light theme: slate-50/white backgrounds, slate-900 text. ReactFlow colorMode follows theme. CSS custom properties on :root.
