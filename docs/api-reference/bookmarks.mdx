## Overview

The Bookmarks API lets you pin any span for later reference, attach investigative notes, and retrieve bookmarks filtered by session. Bookmarks persist in SQLite and survive server restarts.

## Endpoints

### GET /api/bookmarks

List saved bookmarks.

**Query parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `session` | string | Filter to bookmarks whose `traceId` matches this value |

**Response:**
```json
[
  {
    "id": 1,
    "spanId": "abc123def456",
    "traceId": "trace-xyz-789",
    "note": "Suspicious tool call — investigate further",
    "createdAt": "2026-04-08T14:32:01.000Z"
  }
]
```

### POST /api/bookmarks

Create a new bookmark.

**Request body:**
```json
{
  "spanId": "abc123def456",
  "traceId": "trace-xyz-789",
  "note": "Optional free-text note"
}
```

`spanId` is required. `traceId` and `note` are optional.

**Response:** `201 Created` with the created bookmark object.

### PATCH /api/bookmarks/:id

Update the note on an existing bookmark.

**Request body:**
```json
{ "note": "Updated note text" }
```

**Response:** `{ "ok": true }`

### DELETE /api/bookmarks/:id

Delete a bookmark by its numeric ID.

**Response:** `{ "ok": true }` or `404` if not found.

### DELETE /api/bookmarks/span/:spanId

Delete the bookmark for a specific span (toggle-off convenience endpoint).

**Response:** `{ "ok": true }`

## CLI Usage

```bash
# List all bookmarks
claudesec bookmarks

# Filter by session
claudesec bookmarks --session TRACE_ID

# Delete a bookmark by ID
claudesec bookmarks --delete 42
```

## MCP Tools

Two MCP tools expose bookmark functionality to AI agents:

**`bookmark_span`** — Create a bookmark from within an agent:
```json
{
  "name": "bookmark_span",
  "arguments": {
    "spanId": "abc123def456",
    "traceId": "trace-xyz-789",
    "note": "Flagged for manual review"
  }
}
```

**`list_bookmarks`** — Retrieve bookmarks, optionally filtered by session:
```json
{
  "name": "list_bookmarks",
  "arguments": { "traceId": "trace-xyz-789" }
}
```
