## What is a session?

A session corresponds to a single OpenTelemetry trace — all spans sharing the same `traceId` belong to the same session. ClaudeSec automatically creates a session record when it sees a `traceId` for the first time.

The auto-generated session name follows the pattern `{HarnessName} · {time}` (e.g., `Claude Code · 2:34:12 PM`). You can rename sessions via the API or the Sessions panel in the dashboard.

---

## List sessions

```
GET /api/sessions
```

Returns all sessions ordered by creation time, newest first, with aggregate statistics.

### Response

```json
{
  "sessions": [
    {
      "traceId":        "4bf92f3577b34da6a3ce929d0e0e4736",
      "name":           "Claude Code · 2:34:12 PM",
      "createdAt":      "2024-01-15T14:34:12.000Z",
      "spanCount":      47,
      "threatCount":    2,
      "maxSeverityRank": 3,
      "harnesses":      "claude-code"
    }
  ]
}
```

### Session object fields

| Field | Type | Description |
|-------|------|-------------|
| `traceId` | string | OpenTelemetry trace ID — primary key |
| `name` | string | Human-readable session name (editable) |
| `createdAt` | string | ISO 8601 timestamp when the session was first seen |
| `spanCount` | integer | Total number of spans in this session |
| `threatCount` | integer | Number of spans with severity other than `none` |
| `maxSeverityRank` | integer | 0=none, 1=low, 2=medium, 3=high — useful for sorting by worst threat |
| `harnesses` | string | Comma-separated list of distinct harness IDs observed in this session |

## Rename a session

```
PATCH /api/sessions/:traceId
Content-Type: application/json
```

Update the human-readable name of a session.

### Path parameters

| Parameter | Description |
|-----------|-------------|
| `traceId` | The trace ID of the session to rename |

### Request body

```json
{ "name": "My feature branch — Jan 15" }
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | New session name. Must be a non-empty string after trimming whitespace. |

### Response

```json
{ "status": "ok" }
```

### Error responses

| Status | Body | Reason |
|--------|------|--------|
| `400` | `{"error": "name required"}` | `name` was missing or blank |

After a successful rename, a `sessions-update` Socket.io event is broadcast to all connected clients.

## Examples

### List all sessions

```bash
curl http://localhost:3000/api/sessions
```

### Rename a session

```bash
curl -X PATCH http://localhost:3000/api/sessions/4bf92f3577b34da6a3ce929d0e0e4736 \
  -H 'Content-Type: application/json' \
  -d '{"name": "Refactoring session — auth module"}'
```

### Find sessions with high-severity threats

```bash
curl -s http://localhost:3000/api/sessions | \
  jq '.sessions[] | select(.maxSeverityRank == 3)'
```

### Count total sessions

```bash
curl -s http://localhost:3000/api/sessions | jq '.sessions | length'
```
