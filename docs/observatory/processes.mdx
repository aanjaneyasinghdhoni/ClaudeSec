## Overview

Process Monitor scans local process tables (`ps`) for known agent CLIs and exposes controls to terminate, pause, or resume them.

Supported platforms:

- macOS
- Linux

Windows returns no process rows and control endpoints return `501`.

---

## API surface

| Method | Path | Purpose |
|---|---|---|
| `GET` | `/api/processes` | List detected agent processes |
| `DELETE` | `/api/processes/:pid` | Kill one recognized agent process (`SIGTERM`) |
| `POST` | `/api/processes/kill-all` | Kill all detected agent processes |
| `POST` | `/api/processes/pause-all` | Pause all detected agent processes (`SIGSTOP`) |
| `POST` | `/api/processes/resume-all` | Resume all paused agent processes (`SIGCONT`) |

---

## `GET /api/processes` response notes

- `cmd` is username-scrubbed (e.g. `/Users/***`)
- `user` is always `***` (never exposes OS username)
- `recentSessions` attaches sessions from the last 2 hours by harness match

Example:

```json
{
  "processes": [
    {
      "pid": 12345,
      "harness": "claude-code",
      "harnessName": "Claude Code",
      "cmd": "/Users/***/.local/bin/claude ...",
      "cpuPct": 7.4,
      "memMb": 182.3,
      "startedAt": null,
      "user": "***",
      "recentSessions": [{ "traceId": "trace-1", "name": "Claude Code · 2:31 PM" }]
    }
  ],
  "total": 1,
  "scannedAt": "2026-04-18T00:00:00.000Z",
  "supported": true
}
```
