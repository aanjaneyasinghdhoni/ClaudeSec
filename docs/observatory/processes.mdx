## Overview

The **Process Monitor** scans your OS process table every 5 seconds for known AI agent CLI harnesses and displays them in a live-updating table. It answers the question: _"What agents are actually running on my machine right now?"_ — even before they emit any OTLP telemetry.

This is especially useful when you run many agents simultaneously and need a quick overview of resource consumption and activity.

## Supported Platforms

| Platform | Support |
|----------|---------|
| macOS    | ✅ Full support (`ps aux`) |
| Linux    | ✅ Full support (`ps aux`) |
| Windows  | ❌ Not supported |

## Dashboard UI

Navigate to the **Processes** tab to see the live process table.

```
Processes tab → live-updating table (5s poll)
```

Each row shows:
- **Agent** — harness name with color dot
- **PID** — OS process ID
- **User** — the system user running the process
- **CPU** — color-coded bar (green → orange → red as load increases)
- **Memory** — RSS in MB
- **Recent Sessions** — clickable links to sessions from this harness in the last 2 hours
- **Command** — truncated command line

### Summary Cards

Three summary cards at the top show:
- Total agent processes detected
- Combined CPU usage across all agents
- Combined memory footprint

### Auto-refresh

The table polls every 5 seconds by default. Click **Paused** to freeze the view, or **Refresh** to force an immediate scan.

## CLI Usage

```bash
claudesec processes
```

Prints a table of detected agent processes with PID, CPU, memory, and command.

**Example output:**

## API Reference

### GET /api/processes

Returns running agent processes with session correlation.

**Response:**
```json
{
  "processes": [
    {
      "pid": 12345,
      "harness": "claude-code",
      "harnessName": "Claude Code",
      "cmd": "/usr/local/bin/claude --model claude-sonnet-4-6",
      "cpuPct": 12.3,
      "memMb": 248.0,
      "startedAt": null,
      "user": "alice",
      "recentSessions": [
        { "traceId": "abc123", "name": "Claude Code · 2:31:00 PM" }
      ]
    }
  ],
  "total": 1,
  "scannedAt": "2026-04-08T14:32:01.000Z",
  "platform": "darwin",
  "supported": true
}
```

| Field | Type | Description |
|-------|------|-------------|
| `pid` | number | OS process ID |
| `harness` | string | Harness identifier (e.g. `claude-code`) |
| `harnessName` | string | Human-readable harness name |
| `cmd` | string | Command line (truncated to 200 chars) |
| `cpuPct` | number | CPU usage percentage |
| `memMb` | number | Resident set size in MB |
| `user` | string | OS user running the process |
| `recentSessions` | array | Sessions from this harness in the last 2 hours |
| `supported` | boolean | `false` on Windows |

### Session Correlation

Processes are automatically correlated with sessions from the same harness that were active in the **last 2 hours**. Click a session link in the UI to jump directly to that session's graph view.

## Privacy Note

The process scanner only reads publicly available process metadata (`ps aux`) — the same information any user on the system can see. It does **not** read process memory, files, or network connections.
