## Data flow

```
AI Agent Harness
  │
  │  POST /v1/traces  (OTLP JSON over HTTP)
  ▼
server.ts — ingest pipeline
  ├─ Rate limiter (token bucket, 50 RPS / 200 burst per IP)
  ├─ Circuit breaker (503 at 90% of max-span cap)
  ├─ scrub.ts         → redact PII, detect honeytoken hits
  ├─ harnesses.ts     → detect agent type from service.name
  ├─ detectSeverity() → match built-in + custom regex rules
  ├─ SQLite write     → spans · sessions · alerts · spans_fts
  ├─ Behavioral engine → token spikes · threat bursts · off-hours
  ├─ Threshold rules  → numeric metric checks per session
  ├─ Socket.IO emit   → graph:update · span:added · alert:new
  └─ Webhook fire     → Slack / Discord / custom HTTP POST
           │
           ▼
      React UI (App.tsx)
        ├─ ReactFlow graph   — radial layout (default) or dagre
        ├─ Session sidebar   — pin · rename · label · compare · notes
        └─ 11 analysis tabs — see tab list below
```

## Backend — `server.ts`

### SQLite tables

| Table | Purpose |
|---|---|
| `spans` | All ingested OTLP spans — primary data store |
| `sessions` | Trace groupings with name, pinned flag, label, notes |
| `alerts` | Security alert log with deduplication (15-min fingerprint window) |
| `span_bookmarks` | Analyst-bookmarked spans with notes |
| `suppressions` | Time-gated rule snoozes |
| `span_tags` | Custom analyst labels on individual spans |
| `threshold_rules` | User-defined numeric trigger conditions |
| `annotations` | Free-text investigation notes on spans |
| `spans_fts` | FTS5 virtual table — full-text search across names + attributes |
| `webhook_deliveries` | Outbound delivery log (last 500 rows) |
| `config` | Key-value store (webhook URL, retention, honeytokens) |

### Background jobs

| Job | Interval | Description |
|---|---|---|
| Process scanner | Every 30s | Detect running AI agents via `ps` / `tasklist` |
| Auto-export | Every 1h | JSON snapshot to `exports/` (last 24 kept) |
| Span pruning | On ingest | Age-based (30 days) + count-based (50k cap) |
| Delivery log pruning | On webhook fire | Trims to 500 rows |

### Security engine layers

| Layer | Description |
|---|---|
| **Built-in rules** | ~20 regex patterns for secrets, API keys, PII, shell injection |
| **Custom rules** | User-defined regex with severity labels (persisted in `rules.json`) |
| **Threshold rules** | Fire on metrics: `tokens_in`, `tokens_out`, `threat_count`, `span_count`, `high_threat_count` |
| **Behavioral anomaly** | Token spike (4× avg), threat burst (≥3/10 spans), excessive tool calls (>100), off-hours activity |
| **Honeytokens** | Operator-planted canary strings — any match triggers a HIGH severity alert |
| **Alert deduplication** | 15-minute fingerprint window (rule + session + harness) prevents flooding |
| **Suppression cache** | 2-second TTL cache avoids per-span SQLite round-trips |

## Frontend — `src/`

React 19 + TailwindCSS v4 + ReactFlow (`@xyflow/react`) + Socket.IO client + Framer Motion.

### `App.tsx` responsibilities

- **Graph canvas** — ReactFlow with radial layout (threat nodes on outer danger ring) or dagre fallback
- **Session sidebar** — list, pin, rename, label, compare (Ctrl+click), notes drawer
- **Tab routing** — 11 tabs via `activeTab` state (no React Router)
- **Socket.IO** — `graph:update`, `span:added`, `alert:new` events drive live updates
- **Live ticker** — streaming span feed overlay (fades after 10s of quiet)
- **Desktop notifications** — browser `Notification` API for HIGH-severity spans
- **Graph replay** — step-by-step span playback with speed control
- **Theme** — dark / light toggle with `localStorage` persistence

## `scrub.ts` — PII redaction

Runs on every span's attributes before they are written to the database, broadcast over Socket.IO, or forwarded to a remote OTLP collector.

| What is redacted | How |
|---|---|
| Home directory paths | Regex: `/Users/name`, `/home/name`, `C:\Users\name` |
| Process `$HOME` | Dynamic regex from `os.homedir()` |
| OS username | Word-boundary regex from `os.userInfo().username` |
| Email addresses | Local part masked, domain preserved |
| Sensitive key names | Full value replaced with `***` for keys matching `authorization`, `api-key`, `token`, `secret`, etc. |
| Honeytoken strings | Detected in original values before scrubbing; fires HIGH alert |

Disable with `CLAUDESEC_DISABLE_SCRUB=1`.

## Environment variables

| Variable | Default | Purpose |
|---|---|---|
| `PORT` | `3000` | HTTP server port |
| `CLAUDESEC_WEBHOOK_URL` | — | Slack / Discord / custom webhook URL |
| `CLAUDESEC_WEBHOOK_THRESHOLD` | `high` | Minimum severity to fire webhook |
| `CLAUDESEC_DISABLE_SCRUB` | `0` | Set to `1` to disable PII scrubbing |
| `CLAUDESEC_HONEYTOKENS` | — | Comma-separated canary strings |
| `CLAUDESEC_MAX_SPANS` | `50000` | Maximum spans retained |
| `CLAUDESEC_RETENTION_DAYS` | `30` | Days before old sessions are pruned |
| `CLAUDESEC_AUTO_EXPORT_DIR` | `./exports` | Auto-export output directory |
| `CLAUDESEC_OTLP_FORWARD_URL` | — | Forward raw OTLP to another collector |
| `CLAUDESEC_RATE_LIMIT_RPS` | `50` | Sustained ingestion rate per IP |
| `CLAUDESEC_RATE_LIMIT_BURST` | `200` | Burst capacity per IP |
| `CLAUDESEC_MAX_SPANS_BATCH` | `500` | Max spans accepted per OTLP batch |
